# Support

Use the right channel so issues are handled safely and efficiently.

## Questions and Usage Help

Use GitHub Discussions for:

- Installation and configuration questions.
- Usage help.
- Roadmap discussion.
- Integration patterns for Laravel apps.

## Bugs and Feature Requests

Use GitHub Issues for:

- Reproducible bugs.
- Focused feature requests.
- Documentation corrections.

Include package version, PHP version, Laravel version, database driver, and the smallest reproduction you can provide.

## Security Issues

Do not open public issues for suspected vulnerabilities.

Report security issues through GitHub Security Advisories or follow the process in `SECURITY.md`.

## What Not to Share

Never share real:

- JWT signing private keys.
- Access tokens or refresh tokens.
- API keys.
- OAuth client secrets.
- OTPs or password reset tokens.
- `.env` files.
- Customer data.
